NoSQL -- Not Only SQL

definition

"non-relational data stores that do not need a fixed schema, and usually avoid join operations"

emerging categories

  1. key-value stores
    • Similar to associative array
    • Amazon's Dynamo paper
  2. ColumnFamily / BigTable clone
    • large tabular abstraction with column families
    • rows can, in theory, have their own schemas
    • Google's BigTable paper
  3. document databases
    • collections of K-V collections (expressed as JSON)
    • CouchDB/MongoDB (inspired by lotus notes)
  4. graph databases
    • nodes, relationships, K-V on both
    • node is contained in another node, etc

areas of focus

  • scalability to size
  • scalability to complexity
