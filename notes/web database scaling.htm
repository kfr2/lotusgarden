scaling up

  • adding more hardware to a single machine

scaling out

  • adding more machines to db cluster
  • architecture types:
    1. master/slave (http://dl.dropbox.com/u/7030113/Screenshots/0.png)
      • increased read speed
      • takes read off master
      • allows joins across all tables
      • doesn't increase write throughput
      • single point of failure in master node
    2. sharding (http://dl.dropbox.com/u/7030113/Screenshots/1.png)
      • splitting data among obvious boundaries (geographical, etc)
      • increase write and read throughput
      • no single point of failure
      • cannot join queries between shards
